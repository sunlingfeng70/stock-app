# A股盘后分析桌面程序 — 设计规格

## 概述

个人使用的 A 股盘后分析桌面程序。管理 4 只自选股票，基于历史行情数据通过技术指标策略生成买入/加仓/卖出信号，保存信号历史，支持回测验证。

## 技术选型

| 层级 | 选型 | 理由 |
|---|---|---|
| 语言 | Python 3.10+ | 生态成熟，数据分析库丰富 |
| GUI | PySide6 (Qt6) | 跨平台原生外观，组件丰富，文档完善 |
| 数据源 | AKShare | 免费开源，覆盖 A 股全部日K线数据 |
| 技术分析 | pandas + numpy + ta-lib（可选）| 指标计算，ta-lib 为加速选项 |
| 存储 | SQLite (sqlite3) | 零配置，Python 内置，嵌入式 |
| 图表 | matplotlib | PySide6 原生嵌入支持 |

## 策略设计

### 核心指标

| 指标 | 参数默认值 | 用途 |
|---|---|---|
| MA20 | 20 日 | 中期趋势 |
| MA60 | 60 日 | 长期趋势 |
| 布林带 (Bollinger Bands) | 20 日, 2σ | 价格区间判定 |
| RSI | 14 日 | 动量/超买超卖 |
| 成交量变化 | 5 日均量比 | 信号确认 |

### 信号规则

**买入信号（需同时满足 2 项以上）:**
- 收盘价触及或低于布林下轨
- RSI < 30（超卖区域）
- 成交量较 5 日均量放大 > 20%（恐慌性放量见底）
- MA20 走平或向上（非下降趋势）

**加仓信号（需同时满足 2 项以上）:**
- 收盘价站上 MA20
- MA20 向上运行
- 成交量温和放大 > 50日均量
- RSI 在 40-65 之间（非超买区域）

**卖出信号（需同时满足 2 项以上）:**
- 收盘价触及布林上轨或以外
- RSI > 70（超买区域）
- 跌破 MA20 且 MA20 开始走平向下
- 天量滞涨（成交量巨量但涨幅 < 1%）

**仓位建议:**
- 买入: 计划仓位的 50%
- 加仓: 剩余 30%
- 持有观察: 不动
- 卖出: 清仓

所有策略参数（MA 周期、RSI 阈值、布林带系数等）通过 `config.py` 可配置。

## 数据流

```
用户点击"刷新" 或 定时触发
    ↓
AKShare.stock_zh_a_hist() 获取日K线（前复权，最近 365 交易日）
    ↓
pandas 批量计算技术指标
    ↓
策略引擎逐日扫描，对比最新一日与前一日信号
      ├─ 信号变化 → 写入 signals 表
      └─ 信号不变 → 跳过
    ↓
UI 刷新：K线图 + 当前信号 + 信号历史表格
```

## 数据存储 (SQLite)

### 表结构

**`stocks`**
| 列 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PK | 自增主键 |
| code | TEXT UNIQUE | 股票代码，如 '600001' |
| name | TEXT | 股票名称 |
| added_at | TEXT | ISO 日期 |

**`signals`**
| 列 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PK | 自增主键 |
| stock_code | TEXT FK→stocks | 股票代码 |
| date | TEXT | 交易日 |
| signal_type | TEXT | 'buy' / 'add' / 'sell' / 'hold' |
| price | REAL | 当日收盘价 |
| ma20 | REAL | 当日 MA20 |
| ma60 | REAL | 当日 MA60 |
| rsi | REAL | 当日 RSI |
| boll_upper / boll_mid / boll_lower | REAL | 布林带三项 |
| volume_ratio | REAL | 当日量比（相对 5 日均量） |
| note | TEXT | 触发信号的理由文本 |

**`price_cache`**
| 列 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PK | 自增主键 |
| stock_code | TEXT | 股票代码 |
| date | TEXT | 交易日 |
| open / high / low / close | REAL | OHLC |
| volume | INTEGER | 成交量 |

## 非功能需求

1. **数据缓存：** 同一股票同一日期不重复请求 AKShare，优先使用本地缓存
2. **离线可用：** 已有缓存数据可离线查看，刷新时联网
3. **策略可调：** 所有阈值通过 config.py 集中配置，界面提供调节入口
4. **启动速度：** 启动时不自动拉取数据，仅加载本地缓存
5. **信号去重：** 同一天不重复写入相同信号
